# Roadmap

## Agora

- Validar visualmente Dashboard e Transactions em uso real.
- Redesenhar Festometro como gerenciador completo de eventos, mantendo entrada simples e visual mais divertido.
- Verificar se exclusao e edicao de recorrencias se mantem corretas apos reload.
- Revisar textos com encoding quebrado e normalizar copy.
- Revisar Budgets, Analytics, Leisure e Settings com foco em:
  - responsividade desktop
  - simplificacao de cards
  - clareza hierarquica
  - coerencia entre abas
- Criar base compartilhada de layout para reduzir repeticao de estrutura.

## Proximo

- Evoluir Festometro com lista de compras como plus da ferramenta:
  - itens sugeridos por tipo de evento
  - quantidades editaveis
  - status comprado/pendente
  - agrupamento por categoria
  - possibilidade futura de compartilhar lista com participantes
- Aplicar a nova linguagem visual em Budgets, Analytics e Settings.
- Criar componentes de layout compartilhados para cabecalhos, blocos metricos e estados vazios.
- Melhorar feedback de erro nas operacoes com Supabase.

## Depois

- Criar testes para store de financas cobrindo:
  - exclusao simples
  - exclusao futura
  - exclusao da serie inteira
  - atualizacao simples
  - atualizacao em lote
